# 原子化内容管线 (Atomization Pipeline)

> 飞书多维表格作为知识库，资料拆成原子记录，写文章时按选题组合原子。

## 架构

```
Source_Index（资料来源索引，商用状态总控）
    ↓ 关联字段
Atom_Public（行业知识原子库）+ Atom_Company（公司纵向原子库）
    ↓ 配方查询
Topic_Recipe（选题配方与风格表）
    ↓ 按配方组合原子
正文生成（接入 ai-gzh-platform 流程）
```

## 4 张表设计

| 表名 | 用途 | 核心字段 |
|---|---|---|
| 行业知识原子库 | 公开资料拆出的原子 | 原子类型/标题/正文/关键指标/场景标签/来源/商用状态/置信度/图片附件 |
| 公司纵向原子库 | 公司内部资料拆出的原子 | 原子类型/标题/正文/关键指标/业务线/场景标签/来源/商用状态/保密级别/来源文件 |
| 资料来源索引表 | 文档索引+商用状态总控 | 资料名称/来源渠道/类型/文件/商用状态/已拆原子数/关联行业原子/关联公司原子 |
| 选题配方与风格表 | 指导正文生成的配方 | 选题名/方向/目标读者/所需原子类型/推荐数量/风格模板/图片需求 |

## 原子类型分类

### 行业原子（Atom_Public）
- 行业趋势：市场规模、增长率、投融资、产业链结构
- 应用场景：定义 + 痛点 + 无人机价值点
- 技术概念：是什么 + 解决什么问题 + 典型应用
- 政策法规：内容摘要 + 对企业/应用的影响
- 外部案例：项目概况 + 技术方案 + 成果指标
- 职业/培训：证书体系、考试内容、薪资水平

### 公司原子（Atom_Company）
- 公司定位、使命愿景与价值观、资质与荣誉
- 核心团队与人才结构、主营业务场景
- 项目案例、标准与流程、国际合作

## lark-cli 实操要点

### 批量创建记录
```bash
# 格式：fields 是列名数组，rows 是行数组（每行匹配 fields 顺序）
lark-cli base +record-batch-create \
  --base-token <token> --table-id <table_id> \
  --json '{"fields":["原子类型","原子标题"],"rows":[["行业趋势","标题"]]}' \
  --as bot
```

### Select 字段写入
- 单选：直接用文本值 `"行业趋势"`，无需 option ID
- 多选：用数组 `["外立面巡检","光伏运维"]`
- ⚠️ 选项值必须已存在，否则报 `800030005 not_found`

### 附件上传（必须单独操作）
```bash
# 1. 先 cd 到文件目录（--file 只接受相对路径）
cd /path/to/files
# 2. 用字段 ID（不是字段名）上传
lark-cli base +record-upload-attachment \
  --base-token <token> --table-id <table_id> \
  --record-id <record_id> --field-id <field_id> \
  --file ./image.png --as bot
```

### 注意事项
- `--json @file` 只接受相对路径，先 cd
- 附件字段不能在 batch-create 时写入，必须用 `+record-upload-attachment`
- field-id 用真实 ID（如 `fldJPaST93`），不能用字段名，先 `+field-list` 获取
- 批量写入建议逐条调用，防整批失败丢数据
- Select 选项不存在会报 `800030005`，先检查字段选项列表
