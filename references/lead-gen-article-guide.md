# 引流文写作指南（Lead-Gen Article Guide）

> 当用户说「引流/获客/展示服务能力/案例包装/帮企业做XX/写给客户看」时，用本指南替代 n8n-blogger 和 khazix-writer。

## 核心原则

**引流文 ≠ 技术文档。** 读者是企业老板/运营负责人，不关心你用了几个脚本、写了多少行代码。他们关心的是「能不能帮我解决问题」「值不值得花钱找你做」。

**自检**：读者看完会不会想「我也需要这个」？不会 = 还是太技术。

## 用户画像

| 维度 | 画像 |
|---|---|
| 角色 | 中小企业老板、运营负责人、内容团队负责人 |
| 行业 | 有公众号需求的企业（不限行业，偏B端/技术/服务类） |
| 现状 | 有公众号但更新频率低（周更或更低），内容生产靠人工 |
| 痛点 | 公众号是「重要但不紧急」的事，一直拖着不更新 |
| 决策方式 | 看案例、看效果、看能不能快速跑通 |
| 付费意愿 | 愿意花几千块定制一套系统，但需要先看到效果 |

## 风格：kunpeng 白皮书风（克制、咨询感）

参考 `kunpeng-gzh-platform` 的 `references/writing-style-reference.md`。

| 维度 | 要求 |
|---|---|
| 标题 | **场景式/痛点式**，不用数字暴击。≤20字 |
| 开篇 | 痛点切入，一句话建立认知差 |
| 语气 | 克制、咨询感、像白皮书。不用感叹号 |
| 结构 | 8章节（现状→解法→案例→对比→适合谁→交付→角色价值→CTA） |
| 数据 | 真实数据+来源标注。没有就定性表述，不编 |
| 表格 | 每章节1个业务表格（环节×痛点×后果），不是技术表格 |
| 结尾 | 角色价值总结 + 自然CTA，不喊口号 |

### 标题模式（场景式，≤20字）

1. **痛点型**：「公众号停更3个月，不是因为没人写」
2. **场景型**：「1个人管3个公众号，怎么做到的」
3. **反问型**：「公众号内容生产，能不能外包给AI」
4. **对比型**：「你的同行已经在用AI发公众号了」

**不要用**：数字暴击式（「7步搞定AI公众号」）、纯技术式（「1259行代码重构系统」）

### 语气要求

| ✅ 正确 | ❌ 错误 |
|---|---|
| 「公众号的真正问题不是写不出来，是跑不动」 | 「7个Phase替代13步Checklist」 |
| 「帮企业搭了条公众号自动生产线」 | 「1259行代码+5个脚本+42条踩坑」 |
| 「如果你也在为公众号更新频率发愁，可以聊聊」 | 「5个命令搞定」 |
| 克制、咨询感、像白皮书 | 冷峻短句、判断式结论 |


## 配图策略

引流文的配图以**真实截图**为主，AI生成图为辅：

| 类型 | 用途 | 来源 |
|---|---|---|
| 封面图 | 公众号封面 | 归藏 social-card skill 或 GPT Image 2 |
| 真实界面截图 | 展示系统实际运行效果 | 用户提供（飞书/后台/数据面板） |
| 系统架构图 | 展示技术方案 | HTML渲染或代码生成 |
| 信息图 | Before/After对比 | 归藏 material-illustration skill |
| HTML效果截图 | 展示生成的文章效果 | chromium --headless --screenshot |

**截图技巧**（2026-07-11 实测）：
- snap chromium 不能访问 `file://`，必须起 HTTP 服务
- `python3 -m http.server 8777 --bind 127.0.0.1` + `chromium --screenshot --window-size=800,900`
- 截关键部分（标题区/表格区/配图区/CTA区），不用截全页
- 截完 kill HTTP 服务进程

## 禁用词（同 kunpeng 风格）

赋能、闭环、颠覆式、颗粒度、抓手、底层逻辑、值得注意的是、综上所述、首先其次最后、总体而言、不容忽视、唯一、引领、颠覆

## 硬规则（2026-07-13 新增）

### 不提具体客户公司名
引流文是给大姚自己引流用的，不是给客户做宣传。案例部分用泛化表述：
- ✅「一家做技术服务的企业」「我们帮过的一家公司」「某客户」
- ❌「鲲鹏翼航」「XX科技有限公司」
- 写完后全文搜索客户公司名，出现就替换

### 配图必须含真实截图
不能全是AI生成的概念图。至少1张真实截图（飞书界面/公众号后台/HTML渲染效果）。
如果拿不到真实截图，明确告知用户并给出替代方案。
