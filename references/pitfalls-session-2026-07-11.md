# Pitfalls from 2026-07-11 Session

> Append these to the main Pitfalls list in SKILL.md as #53-55.

## 53. build_html.py IMG_KEYWORDS 缺少高编号配图

**现象**：文章有6个配图标记，但HTML只内嵌了5张（封面+配图1-4）。配图5和配图6被静默跳过。

**根因**：`scripts/build_html.py` 的 `IMG_KEYWORDS` 字典只预定义了配图1-4的关键词。配图5和配图6的 `kw_list` 返回空列表，编号匹配和关键词匹配都失败。

**修法**：已修复——IMG_KEYWORDS 增加了配图5/6的条目。配图5关键词：`["配图-5", "配图5", "html", "效果", "preview"]`。配图6关键词：`["配图-6", "配图6", "before-after", "before", "after"]`。

**防再犯**：未来配图超过6张时，必须同步扩展 IMG_KEYWORDS 字典。建议在 build_html.py 的注释中加一行提示：`# 新增配图编号时必须同步添加关键词条目`。

## 54. 图片文件名不含配图编号导致嵌入失败

**现象**：图片文件名如 `飞书素材库截图.jpg` 或 `系统架构图.png`，不含「配图1」「配图2」等编号，build_html.py 无法匹配。

**根因**：embed_image 函数先按编号精确匹配（从 marker「配图1」提取数字1，查找文件名含「配图1」的文件）。文件名不含编号时编号匹配失败，再按关键词模糊匹配，但 IMG_KEYWORDS 的关键词列表可能也不覆盖该文件名。

**修法**：文件名必须以 `配图N-` 开头（N=1,2,3...），后面接中文关键词。格式：`配图N-<中文关键词>-<英文slug>.png`。

**正确示例**：
- `配图1-飞书素材库-行业原子库.jpg` ✓
- `配图2-系统架构图.png` ✓
- `配图5-案例数据-html.png` ✓

**错误示例**：
- `飞书素材库截图.jpg` ✗（不含编号）
- `arch-diagram.png` ✗（不含中文关键词和编号）

## 55. 引流文风格路由触发条件

**现象**：用户说「帮我写一篇公众号内容」，agent 按 n8n-blogger 技术教程风格写，用户否了（「啥玩意 不行 太差了 效果」）。

**根因**：没区分「文章类型」。用户上下文涉及「帮XX公司做了XX服务」时，意图是引流文/服务展示，不是技术教程。

**判断规则**：
- 用户说「引流/获客/展示服务能力/案例包装/帮企业做XX/写给客户看」→ 引流文
- 用户说「写公众号内容」+ 上下文有「帮XX公司做了」→ 引流文
- 用户说「写技术文章/记录改造过程」→ 技术文
- 用户说「按SOP写内容/生产今天的公众号」→ 标准内容文

**引流文必须**：用 `references/lead-gen-article-guide.md`，不用 n8n-blogger 或 khazix-writer。自检：读者看完会不会想「我也需要这个」？

## 56. 用户发的HTML文件要截图替换配图

**现象**：用户发来一个HTML文件说「里面的那张html截图搞错了 应该是这份」。
**修法**：收到用户指定的HTML文件后，不要用「已经截好了」跳过。正确流程：
1. 复制到 output 目录
2. 起 HTTP 服务（`python3 -m http.server 8779 --bind 127.0.0.1`，用不同端口避免冲突）
3. chromium 截图（URL中文文件名先 `cp` 成英文名再访问）
4. 替换对应配图文件
5. 重新生成 HTML + 发送

## 59. GPT Image 2 API 中文文字 prompt 超时频率高（2026-07-13 实测）

**现象**：用 `generate_image.py` 或直接 `requests.post` 调 GPT Image 2 API，prompt 含中文标签文字时，120-180s 超时率极高。本次 session 6 次超时（含 3 次 `generate_image.py` + 3 次直接调用），仅 2 次成功。

**规律**：
- 纯英文 prompt + 简单概念 → 通常 60-90s 成功
- 含中文标签（如 `行业趋势/应用场景/技术概念`）→ 超时概率 >70%
- `1792x768`（封面）比 `1792x1024`（正文图）更容易超时
- `generate_image.py` 内置 3 次重试，但 3 次全超时后仍失败

**降级策略**（按优先级）：
1. **先用已有图交付 HTML**：2 张图也能看效果，不阻塞交付节奏
2. **delegate_task 后台生成**：把剩余图生成任务交给子代理，不阻塞主流程
3. **简化 prompt**：去掉中文标签，用英文 `Industry Trend / Application Scenario / Tech Concept` 替代，超时率显著降低
4. **分批生成**：一次只生 1 张，不并行——并行请求更容易全部超时
5. **绝不用旧选题的图冒充**（Pitfall #31 已禁）

**自检**：生成图片后检查文件大小。正常 500KB-1.5MB。如果 <100KB，可能是截断的空图，删掉重生成。

## 60. 引流文系列写作：角度差异化检查表（2026-07-13 实测）

**场景**：用户说「前天写的引流效果不错，再写一篇不同角度的」。需要写同主题但不同切入角度的引流文，避免读者觉得「又是老一套」。

**角度差异化维度**：

| 维度 | 前天（7/11） | 今天（7/13） | 未来可选 |
|---|---|---|---|
| **核心痛点** | 流程太长、环节太多 | 每次写稿从零搜素材 | 编辑离职=公众号停更 |
| **解法角度** | 流程脚本化（How） | 素材库建设（What） | 一人多号运营（Who） |
| **案例展开** | 全流程改造效果 | 素材库4步建设法 | 内容资产化ROI |
| **Before/After** | 4环节→1环节 | 无素材库→有素材库 | 养编辑 vs 搭产线 |
| **角色价值** | 省人/省心/省力 | 资产化/可预期/专注 | 成本可控/产能翻倍 |
| **标题模式** | 帮企业做了XX | XX为什么总是YY | 你的同行已经在XX |

**写之前必做**：
1. 回顾最近 2-3 篇引流文的标题、角度、CTA
2. 从上表选一个**未使用过的组合**
3. 自检：换个标题和开头，正文结构能不能复用？能=角度太近，需要换

**CTA 也要差异化**：
- 前天：「聊完给你一个方案和报价」
- 今天：「素材梳理和结构化也是服务的一部分」
- 未来：「回复 XX 获取案例集」

## 61. 引流文不提具体客户公司名（2026-07-13 用户纠正）

**现象**：用户说「鲲鹏翼航哪里不要提这个公司」。引流文是给大姚自己引流用的，不是给客户做宣传。

**修法**：用泛化表述——「一家做技术服务的企业」「我们帮过的一家公司」「某客户」。除非用户明确授权可以提具体公司名。

**自检**：写完引流文后全文搜索客户公司名，出现就替换为泛化表述。

## 62. 用户要「真实图片」时不要给AI生成概念图（2026-07-13 用户纠正）

**现象**：用户说「里面一定要有真实图片」。AI生成的3D插图是概念图，不是「真实图片」。

**真实图片定义**：飞书界面截图、公众号后台截图、HTML渲染效果截图、实际数据面板截图。不是AI生成的3D材质插画。

**降级策略**（如果拿不到真实截图）：
1. 明确告知用户「目前是AI概念图，真实截图需要你提供XXX」
2. 给出替代方案：用户截图发过来我嵌入 / 用已有HTML文章截图 / 用飞书Base数据生成表格样式图
3. 用 chromium --headless 截图已有的公众号HTML文章作为「效果展示」图

**❌ 不要默认AI图=真实图**。配图可以是AI概念图+真实截图混搭，但必须有至少1张真实截图。
