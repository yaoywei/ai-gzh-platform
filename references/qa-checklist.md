# QA Checklist

## 必过项
- 是 16:9 横版。
- 背景是干净白底。
- 有小姚。
- 小姚承担核心动作，不只是装饰。
- 小姚有清晰识别点：方形眼镜、炸毛短发、橙色卫衣、深灰裤子。
- 没有复刻旧案例构图，而是为当前文章生成了新隐喻。
- 画面清爽、有创意、有意思。
- 简洁清楚，主体不超过画面约 60%。
- 一张图只讲一个核心结构。
- 中文标注少、短、能读。
- 暖橙色主要用于小姚和重点/箭头。
- 深灰色主要用于小姚服装、系统组件或补充说明。
- 红色只用于重点、问题、提醒或风险。

## 失败信号
出现以下情况，重生成或局部编辑：
- 左上角有"常见坑 / Workflow / 系统架构图 / 路线图"等标题。
- 小姚像普通二次元帅哥、偶像角色、儿童角色或酷炫摆拍。
- 小姚关键特征缺失：没有方形眼镜、没有炸毛短发、不是橙色卫衣深灰裤子。
- 小姚只是站在旁边微笑，没有参与核心动作。
- 画面像 PPT、课程课件、正式流程图。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 背景有纸纹、阴影、渐变、米色、噪点。
- 真实 UI 截图或科技感界面。
- 中文错字严重或标注不可读。
- 画面太死板，没有清爽的隐喻或执行感。
- 和 `assets/examples/` 里的旧案例构图过于相似。

## 迭代方法
- 太普通：让小姚成为动作主体，加入一个清爽但有记忆点的隐喻。
- 太复杂：删节点，只保留一个动作和 3-5 个短标注。
- 太可爱：强调 product architect, automation builder, clean hand-drawn, not chibi, not mascot, not idol。
- IP 不稳定：重复强调 square-frame glasses, spiky short black hair, warm-orange hoodie #F97316, dark grey pants。
- 太 PPT：去掉标题、边框、整齐网格和过多箭头，改成手绘场景。
- 太像旧案例：保留核心意思，换掉主物件和小姚动作。
- 文字错：优先局部编辑；错得多就重生成并减少标注数量。

## 交付判断
高质量图应该让读者先看到一个清爽、务实、有执行感的小姚正在搭建系统，然后 1 秒内看懂结构。

如果第一眼像教程页、商业海报或普通二次元角色图，而不是白纸上的中文正文解释草图，就不合格。
