# 信息峰值密度规则

> 目标：每 300 字至少 1 个信息峰值，让读者「划不走」。

---

## 什么是信息峰值

信息峰值 = 让读者停下来细读的内容锚点。不是过渡句，不是背景铺垫，是**有信息增量**的点。

5 种信息峰值类型：

| 类型 | 说明 | 示例 |
|---|---|---|
| **金句** | 一句话浓缩核心观点，有记忆点 | 「工具不是选题，任务才是选题」 |
| **数据** | 具体数字，有冲击力 | 「选题从 2 小时压到 5 分钟」 |
| **小案例** | 1-2 句真实场景/踩坑 | 「第一次写引流文，堆了脚本行数，用户直接否了」 |
| **对比** | 前后/正反/新旧的反差 | 「以前 1 个人管 1 个号，现在 1 个人管 3 个号」 |
| **反常识** | 打破读者预期的判断 | 「选题池里 200 条选题，你还是不知道写什么」 |

---

## 怎么用

### 写作时

写完一段后自检：这段 300 字里有没有上述 5 种之一？

- 有 → 继续
- 没有 → 补一个：
  - 加一句金句（用粗体标出）
  - 塞一个具体数字
  - 插一个 1-2 句的小案例
  - 加一组前后对比

### 排版时

信息峰值用排版手段强化可见性：
- **金句** → 加粗或引用块
- **数据** → 加粗数字部分
- **小案例** → 前面加 ❌/✅/📝 标记
- **对比** → 用表格
- **反常识** → 用引用块

---

## 密度标准

| 文章长度 | 最少峰值数 | 说明 |
|---|---|---|
| 2500 字 | ≥8 个 | 每 ~300 字 1 个 |
| 3000 字 | ≥10 个 | |
| 4000 字 | ≥13 个 | |
| 5000 字 | ≥16 个 | |

**分布要求**：
- 开头 200 字内至少 1 个（钩子）
- 每个章节至少 2 个
- 结尾 300 字内至少 1 个（回扣）

## 与 postflight 的集成

postflight.py 会自动检查：
1. 段落行距：正文段落每段 ≤ 4 行（手机阅读友好）
2. 信息峰值密度：每 300 字 ≥ 1 个峰值标记

### postflight 识别的峰值标记

| 标记 | 说明 |
|---|---|
| `**粗体句子**` | 可能是金句/数据强调 |
| `「引用内容」` | 可能是金句/反常识 |
| `> 引用块` | 可能是金句/反常识 |
| `\d+%` / `\d+分钟` / `\d+小时` | 数据峰值 |
| `❌` / `✅` / `📝` | 案例/踩坑标记 |
| 表格行 | 对比峰值 |

postflight 的检查是**辅助提醒**，不是硬性阻断。密度不足时输出 ⚠️ 警告但不 block 交付。

## 反模式

| 问题 | 表现 | 修复 |
|---|---|---|
| 峰值全堆在开头 | 前 500 字很精彩，后面平铺直叙 | 均匀分布，每个章节至少 2 个 |
| 峰值都是数据 | 全文只有数字，没有温度 | 交替使用 5 种类型 |
| 假峰值 | 「众所周知」「不言而喻」不是峰值 | 删掉，换成有信息增量的内容 |
| 峰值太密 | 每句话都加粗 = 没有重点 | 保持 ~300 字 1 个的节奏 |
