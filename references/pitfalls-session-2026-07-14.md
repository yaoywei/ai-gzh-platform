# Pitfalls from 2026-07-14 Session

> Append to main Pitfalls list in SKILL.md as #63-66.

## 63. 引流文 vs 标准内容文——风格路由必须准确（2026-07-14 用户纠正）

**现象**：用户说"写今天的公众号内容"，agent用默认的dayao-writing-prompt.md（实战复盘体）写，用户说"这篇不是用的我的那个引流的写的吗"。

**根因**：没判断文章类型。用户最近的公众号文章（7/11、7/13）都是引流文风格，"写今天的公众号内容"在这个上下文中=引流文。

**判断规则**（增强版）：
- 最近2-3篇都是引流文 → 新文章默认引流文
- 用户说"写公众号内容"且没明确说"技术文/复盘" → 默认引流文
- 用户说"写引流文/展示服务/帮企业做XX" → 引流文
- 用户说"写技术文章/记录改造过程" → 技术文

**修法**：写之前先看最近几篇的风格，保持一致。不确定时用clarify问一句。

## 64. 配图必须用归藏材质插画风格（2026-07-14 用户纠正）

**现象**：用普通GPT Image 2 prompt生成配图，用户说"生图的风格怎么也被修改了 按照之前的生图风格来啊 归藏的"。

**根因**：生成配图时没有读取guizang-material-illustration skill，用了通用prompt。

**规则**：公众号配图**统一使用归藏材质插画风格**。生成配图前必须：
1. 读取 `guizang-material-illustration` skill
2. 读取 `references/visual-style.md`（视觉规范）
3. 读取 `references/prompt-patterns.md`（prompt模板）
4. 按模板写prompt，不要自由发挥

**归藏风格要素**：
- Off-white studio background
- Black ink outlines + refined gray surfaces
- One vivid accent color（IKB蓝#002FA7或安全橙#FF6B35）
- 3-5个中文标签作为callouts
- Pipeline/Cycle/Hub/Before-After等视觉结构

## 65. 模拟截图不行，必须真实系统截图（2026-07-14 用户纠正）

**现象**：用HTML+CSS模拟飞书表格界面作为"真实截图"，用户说"这个真实截图效果不行"。

**正确做法**：
1. HTML渲染效果 → `python3 -m http.server 8777` + Playwright截图（中文文件名先cp成英文名）
2. 飞书界面 → 浏览器直接访问截图
3. 已渲染的卡片 → 直接用PNG或拼图

**关键**：模拟的再像也不是"真实"的。真实截图=截取实际运行的系统。

## 66. 引流文正文质量要对齐历史最佳（2026-07-14 用户纠正）

**现象**：用户说"正文内容好像没有7月11号哪篇写的好哦 你看看是提示词 还是哪里变了"。

**根因**：新文章没有参考历史最佳文章的具体写法——7/11那篇有"5000+素材原子""24个选题方向""42条踩坑"等具体数字，有鲲鹏翼航完整案例，有6张真实截图。新文章太泛泛。

**修法**：写引流文之前，先读最近2-3篇的article.md，学习：
1. 具体数字（素材量、选题数、踩坑数）
2. 案例展开方式（真实客户/项目细节）
3. 交付清单（定制周期、售后支持）
4. 角色价值总结（三类人分别看什么价值）

**自检**：写完后和历史文章对比——数字够不够具体？案例够不够真实？交付清单有没有定制周期？
