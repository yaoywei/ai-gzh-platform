# Enterprise Windvane Mode｜AI企业应用风向标结构增强

用于 `ai-gzh-platform` 的 B 端 AI 企业应用公众号内容。它来自「跨境风向标」122 篇有效正文样本的方法论拆解，但只迁移结构，不迁移灰黑业务动作，也不冒充原作者。

## 什么时候启用

用户或选题出现这些信号时启用 `enterprise-windvane`：

- AI企业应用、B端、甲方、决策者、SaaS、Agent落地、企业知识库
- AI工具采购、供应商选择、POC、ROI、预算、回收周期
- 客服/销售/运营/招聘/财务/法务等企业流程自动化
- 用户要求文章更锋利、更有转化、更像销售漏斗，但仍面向企业读者

不用于：C端副业、跨境套利、灰帽账号、纯教程、纯产品 PR、官方新闻稿。

## 核心原则

学结构，不学刺激词。

`enterprise-windvane` 不是让文章变成「暴利/月入万刀/垃圾/割韭菜」口吻，而是让文章具备：

1. 标题有具体场景和冲突。
2. 第一屏下判断。
3. 正文指出大多数企业错在哪里。
4. 用数据、案例、表格和 SOP 证明。
5. 结尾只有一个承接动作。

## 7维选题评分

在原 `ai-gzh-platform` 五维评分基础上，加两项。每项 1-5 分，低于 4.0 不进入正式写作。

| 维度 | 问题 | 权重建议 |
|---|---|---:|
| 生存焦虑 | 不解决会不会造成预算、效率、组织或竞争损失？ | 20% |
| 需求具体 | 是否落在一个岗位、流程、工具、采购或业务场景？ | 15% |
| 可复制 | 读者看完是否能做 1-3 个动作？ | 15% |
| 画像匹配 | 是否匹配老板、业务负责人、AI负责人、销售/客服/运营负责人？ | 10% |
| 数据支撑 | 是否有真实数字、案例、报价、流程指标或报告来源？ | 15% |
| 冲突张力 | 是否能指出「大多数企业判断错了什么」？ | 15% |
| 转化承接 | 是否能自然导向清单、案例包、诊断、工作流模板或咨询？ | 10% |

## 标题公式

标题必须包含「具体场景 + 数字/损失/结果 + 反常识判断」中的至少两项。

可用模板：

1. `月花 <金额> 买 AI 工具，老板们踩的不是技术坑，是 <流程/组织/采购> 坑`
2. `一个 <Agent/系统> 替不了 <人数>，真正省钱的是这 <数字> 个流程节点`
3. `<岗位/部门> 上 AI 失败，通常不是模型差，是 <隐藏变量> 没人管`
4. `<场景> 的 ROI 怎么算？我把账拆到 <指标/周期> 给你看`
5. `别急着买 <工具类别>，先问供应商这 <数字> 个问题`
6. `<行业/部门> 做 AI 试点，最容易浪费钱的 <数字> 个动作`

## 开头模板

### A. 先给结论

```text
先说结论，企业买 AI 工具最容易亏钱的地方，不是模型不够强。

是采购流程还停在买传统软件那一套。
```

### B. 先给数字

```text
一家公司每个月花 50 万买 AI 工具。

三个月后，客服响应时间没降，人工排班也没少。

问题不在工具，在验收指标从第一天就写错了。
```

### C. 先给反常识

```text
很多老板以为，上 AI 的第一步是选模型。

错了。

第一步是找出哪个流程值得被机器接管。
```

## 正文结构

按这个顺序写：

1. **现象**：企业正在为什么花钱、焦虑或试点失败。
2. **错因**：大多数人把问题看成什么，为什么错。
3. **真正变量**：影响结果的 2-4 个变量，例如流程边界、数据质量、验收指标、组织协同、供应商锁定。
4. **对照表**：错误做法 vs 正确做法 vs 业务影响 vs 验证指标。
5. **SOP**：读者今天能做的 3 步。
6. **风险边界**：哪些场景不适合上 AI，或什么时候该停。
7. **单一 CTA**：领取清单、看案例包、预约诊断、获取工作流模板，四选一。

## B端降噪词典

| 不用 | 改为 |
|---|---|
| 暴利 | 高 ROI / 低边际成本 |
| 月入万刀 | 回收周期 / 单人产出 / 成本节省 |
| 割韭菜 | 预算浪费 / 供应商锁定 / 试点失败 |
| 垃圾 | 不适合生产环境 / 无法通过验收 |
| 躺赚 | 自动化节省人工 / 持续运行 |
| 白嫖 | 低成本试点 / 免费额度验证 |
| 打脸同行 | 推翻错误采购假设 |
| 包回本 | 阶段验收 / 效果对赌 / 退出机制 |

## Prompt 适配器

写作时把这一段附加到 MiniMax 写文 system prompt 或用户 prompt：

```text
启用 enterprise-windvane 模式。

你要使用「跨境风向标」拆解出的结构，但必须面向 AI 企业应用读者降噪。
不要模仿原作者身份，不要写灰帽套利，不要使用 C 端刺激词。

写作要求：
1. 标题包含具体企业场景 + 数字/损失/结果 + 反常识判断中的至少两项。
2. 开头 80 个汉字内给出核心结论。
3. 正文必须指出大多数企业错在哪里。
4. 必须有一张对照表，列：错误做法 / 正确做法 / 业务影响 / 验证指标。
5. 至少 3 个证据点，必须来自输入素材、知识库 atoms、报告数字、企业案例或真实工具截图描述。
6. 结尾只允许一个 CTA：资料包、案例包、诊断、工作流模板四选一。
7. 禁止词：暴利、月入万刀、割韭菜、垃圾、躺赚、白嫖。
8. 使用 B 端替代表达：预算浪费、试点失败、ROI、回收周期、供应商风险、组织成本、阶段验收。
```

## 风向标结构自检

在 `L1-L4` 后增加本层。任一项失败，回到正文修改；通过后才能继续配图、HTML 或推草稿。

| 检查项 | 通过标准 |
|---|---|
| 标题钩子 | 有具体场景，并包含数字/损失/结果/反常识之一 |
| 第一屏判断 | 前 80 个汉字内给出明确结论 |
| 错因指出 | 明确写出「大多数企业/老板/团队错在哪里」 |
| 对照表 | 至少 1 张表，含错误做法、正确做法、业务影响、验证指标 |
| 证据点 | 至少 3 个真实证据点，不能编造数字 |
| 单一 CTA | 全文只有一个主要承接动作，且和正文承诺一致 |
| B端降噪 | 禁止词零命中，表达改为企业语境 |

## 常见错误

| 错误 | 修法 |
|---|---|
| 只学强语气 | 回到证据、对照表、SOP，不靠骂人撑文章。 |
| 写成企业白皮书 | 第一屏必须有判断，别从宏观背景开始。 |
| 硬塞 ROI 数字 | 没有来源就写成计算框架，不编百分比。 |
| 多个 CTA | 只保留最贴近正文承诺的一个。 |
| C端词没降噪 | 用 B端降噪词典替换。 |
