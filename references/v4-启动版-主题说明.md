## AGPL 合规说明

**关键边界**：
- 大姚青主题库**只留在摸鱼小李 skill 里**（`~/.claude/skills/gzh-design-skill/references/theme-dayao-qing-clone.md`）
- ai-gzh skill（MIT）**不包含**主题库代码
- 师傅写文章时**调用**摸鱼小李生成 HTML，不 fork 不修改
- ai-gzh skill 仍是 MIT 协议

**为什么这样设计**：
- 摸鱼小李原 skill 是 AGPL-3.0（衍生品必须开源 + 网络服务也要开源 + 不允许闭源专有化）
- 如果把大姚青主题库复制到 ai-gzh skill，整个 ai-gzh 就被传染 AGPL
- 隔离方案让师傅既能享受大姚青的视觉，又保持 ai-gzh 的 MIT 自由度

## 师傅日常使用流程

### 写新文章

1. 选题：vault 60 条精选里挑一条
2. 起草：按 v4.1 5 段式（卡→拆→工具→结果和坑→读者能拿走）
3. 字数：1500-1800 字（v4 区间）
4. 风格：避禁用清单（⭐/双引号/三段式加粗/AI 词汇/公式化反问）
5. 调摸鱼小李生成 HTML：
   ```bash
   npx gzh-design-skill render --input article.md --theme dayao-qing-clone
   ```
6. 用 wrap_preview.py 加复制按钮预览
7. 粘到公众号后台 → 发布
8. 09 库记录：跑 `python3 scripts/cimi_sync.py article-stats <URL>`
