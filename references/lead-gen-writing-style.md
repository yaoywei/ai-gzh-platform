# 引流文写作风格（咨询感白皮书风）

> 来源：鲲鹏翼航公众号内容生产平台的写作风格。适用于「服务展示/案例引流」类文章。
> 触发：用户说「写引流文 / 服务展示 / 案例说明 / 帮我拉客户」时使用。

## 风格特征

### 标题
- **场景式/观点式**：不用数字暴击，用场景痛点或观点
- **模式**：`帮企业做了XX` / `XX不是YY，是ZZ` / `你的XX为什么ZZ`
- **示例**：
  - ✅ 帮企业搭了条公众号自动生产线
  - ✅ 公众号不是写不出来，是跑不动
  - ✅ 你公司的公众号，多久没更新了
  - ❌ 7步搞定AI公众号（数字暴击，偏引流教程）
  - ❌ AI革命来了（纯口号）

### 开篇（1段）
- **痛点切入**，一句话建立「我知道你的问题」
- 示例：
  - 「一个号一天发1-2篇，2-3个号就要一个人全职盯。」
  - 「公众号更新是「重要但不紧急」的事，一直拖着不更新。」
- **不要**：堆砌背景、重复标题、用「随着...的发展」

### 正文结构（8章节）

```
# {主标题}
{副标题一句话}

开篇：痛点切入

---

## 一、现状：大部分企业卡在哪
表格：环节 × 耗时/痛点/后果

## 二、解法：核心观点 + 技术方案
核心观点一段话，然后展开技术方案

## 三、案例：实际效果
真实素材截图 + 数据

## 四、对比：改造前 vs 改造后
表格：维度/旧/新/差距

## 五、适合谁 / 不适合谁
边界清晰

## 六、交付清单
表格：交付物/说明

## 七、角色价值总结
对不同角色分别讲价值

## 八、一句话 + CTA
自然收尾，不硬推
```

### 语气
- **克制**：不用感叹号（除结尾）
- **咨询感**：像白皮书/咨询报告，不像销售话术
- **专业**：用行业术语但不卖弄
- **有边界**：不夸大、不承诺做不到的事
- **不喊口号**：不用「颠覆/引领/唯一」

### 数据使用
- 用**真实数据**，没有就定性表述
- 数据来源标注
- 不编造百分比/工时/转化率

### 禁词
赋能、闭环、颠覆式、颗粒度、抓手、底层逻辑、
值得注意的是、综上所述、首先其次最后、总体而言、不容忽视、
说白了、意味着什么、本质上、换句话说、不可否认

### 禁用标点
- 双引号"" → 用「」代替
- 感叹号（除结尾）

### 结尾
- 角色价值总结，每个角色给具体价值点
- CTA 自然带出，不硬推
- 示例：「如果你也在为XX发愁，可以聊聊。不收咨询费，聊完给你一个方案和报价，你再决定要不要做。」

## 内容丰富度标准

| 要求 | 最低标准 |
|---|---|
| 总字数 | ≥2500字 |
| 数据表格 | ≥2个 |
| 配图标记 | ≥3个 |
| 列表 | ≥1个 |
| 数据引用 | ≥5个具体数字 |

## 与 n8n-blogger 风格的区别

| 维度 | 引流文（本风格）| n8n-blogger |
|---|---|---|
| 读者 | 中小企业老板 | 技术决策者/运营 |
| 标题 | 场景式/观点式 | 数字暴击式 |
| 开篇 | 痛点切入 | 数据暴击 |
| 语气 | 克制、咨询感 | 冷峻、判断式 |
| 结尾 | 角色价值+自然CTA | ABCD选择题 |
| 适用 | 服务展示/案例引流 | 教程/lead-gen/爆文 |

## 硬规则（2026-07-13 新增）

### 不提具体客户公司名
引流文是给大姚自己引流用的，不是给客户做宣传。案例部分用泛化表述：
- ✅「一家做技术服务的企业」「我们帮过的一家公司」「某客户」
- ❌「鲲鹏翼航」「XX科技有限公司」
- 除非用户明确授权可以提具体公司名

### 配图必须含真实截图
引流文配图不能全是AI生成的概念图。至少1张真实截图：
- ✅ 飞书界面截图、公众号后台截图、HTML渲染效果截图、实际数据面板
- ❌ 全是AI生成的3D材质插画
- 降级方案：用已有HTML文章的chromium截图作为「效果展示」图
