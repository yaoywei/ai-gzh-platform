# 真实素材调研方法论（写前必做）

> 用户原话（2026-07-08）："我觉得现在写出来的内容缺乏真实性 没什么干货"
> 用户原话："你可以去我的知识星球找啊 然后你可以用次幂数据 找哪些真实案例的文章啊 也可以去别的平台找啊 多的是方法"

## 核心原则

**先挖真实素材，再动笔写文。** 没有真实工具名、真实成本、真实案例、真实流程的文章 = 废稿。

用户能一眼看出"这是AI编的概括"还是"这是做过的人在聊经验"。区别在于：
- ❌ AI概括：「用开源框架搭建自动化系统」「大语言模型改写内容」
- ✅ 真实经验：「魔云腾云机环境+影刀AI魔法指令3.0+知云矩阵助手」

## 调研四线并行

### 线1：次幂数据公众号（cimi API）

**用途**：找真实公众号文章，提取工具名、数据点、案例。

**流程**：
1. `POST /api/v2/token` 获取 access_token
2. `POST /api/v3/accounts/search` 搜索行业账号（多关键词覆盖）
3. `POST /api/v2/articles/history` 拉文章列表（注意：响应在 `data.items`，不是 `data.articles`）
4. 从标题+摘要提取真实数据点（工具名、成本、数字、流程步骤）

**关键坑**：
- v3 articles/detail 返回500 → 用 v2 也可能被微信反爬（返回验证页面HTML）
- **降级策略**：标题+摘要已经包含足够多的工具名和数据点，不一定需要全文
- 搜索关键词要多轮覆盖：行业词 + 工具词 + 案例词 + 教程词

**搜索关键词模板**：
```
第一轮（行业）：{行业}获客, {行业}引流, {行业}运营
第二轮（工具）：{行业}矩阵工具, {行业}自动化工具
第三轮（案例）：{行业}实操教程, {行业}案例
```

### 线2：知识星球（zsxq-cli）

**用途**：找深度实操帖、真实经验分享、内部教程。

**流程**：
1. `zsxq-cli group +list` 列出所有加入的星球
2. 选择相关星球，`zsxq-cli topic +search --group-id {id} --query "{关键词}"`
3. 读取帖子详情，提取真实工具/流程/数据

**关键坑**：
- 部分星球未开通 Skill 权限 → 降级到 web_search 搜星球外泄内容
- 搜索语法：`--query` 不是 `--keyword`

### 线3：B站/知乎/小红书（web_search + web_extract）

**用途**：找视频教程描述、专栏文章、实操帖子。

**搜索模板**：
```
"{关键词}" 实操教程 工具 真实案例 {年份}
"{关键词}" site:bilibili.com OR site:zhuanlan.zhihu.com OR site:xiaohongshu.com
```

**关键坑**：
- 知乎文章 web_extract 经常失败 → 用 web_search 的 description 提取摘要
- B站视频描述包含丰富的工具名和时间戳 → 优先提取
- 知乎专栏搜索结果的 description 通常包含核心观点

### 线4：用户知识库/历史素材

**用途**：复用用户已有的真实素材。

**检查项**：
- 用户之前发过的文章/截图/数据
- 用户知识星球中的帖子
- 用户历史对话中提到的工具/流程/数字

## 素材提炼标准

从调研中提取的素材必须满足：

| 维度 | 要求 | 示例 |
|---|---|---|
| 工具名 | 具体产品名，不是「开源框架」「AI工具」 | 魔云腾、影刀、知云矩阵助手 |
| 成本 | 有来源的数字，不是编的 | KOS认证600元/年（知乎/2026新规） |
| 流程 | 有具体步骤和参数 | 新号注册→秒评秒私→养3-5天→发笔记 |
| 案例 | 有来源+有结果 | 火锅店7天引流1300人（次幂数据） |
| 踩坑 | 来自真实废号/失败经验 | 废掉500+号后整理的SOP（李子冉） |

## 调研→写作衔接

调研完成后，输出一份 `real-sources.md` 到 output 目录，格式：

```markdown
# 真实素材库

## 来源1：{账号名}（{平台}，{背景}）
### 核心数据点（来自文章标题/摘要）
- {数据点1}
- {数据点2}
### 具体工具名
- {工具1}：{用途}
- {工具2}：{用途}

## 来源2：...

## 提炼的真实干货
### 真实工具名矩阵
| 工具 | 用途 | 来源 |
### 真实成本数据
### 真实流程
```

**然后**把这份素材喂给 Writer prompt 写文章。素材密度 > prompt 长度。

## 自检：文章是否够真实

写完后过这个清单：

- [ ] 文章中是否至少有3个具体工具/产品名？（不是「开源框架」「AI工具」）
- [ ] 所有数字是否标注来源？（不是编的百分比）
- [ ] 是否有至少1个真实案例/人物/团队？（不是「某团队」「某公司」）
- [ ] 流程步骤是否有具体参数？（不是「适当配置」「合理设置」）
- [ ] 踩坑清单是否来自真实经验？（不是AI生成的通用建议）
- [ ] 读者看完能不能直接去搜某个工具名/加某个号？（这是干货的终极检验）
