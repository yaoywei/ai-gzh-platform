# 小姚 IP

## 角色定义
小姚 / Xiao Yao 是中文正文配图的固定视觉 IP。

默认每张图都要出现小姚。小姚不是站在角落里的讲解员，不是酷帅摆拍角色，也不是普通职场男生，而是正在认真搭建系统、拆解模块、连接工具、验证产出的产品架构师/AI自动化搭建者。

## 外形
- 年轻男性，产品架构师 / AI 自动化搭建者。
- 黑色短发，略微炸毛（前额几根翘起），是重要识别点。
- 黑色方形框眼镜，镜框清晰棱角分明。
- 暖橙色卫衣 #F97316，清爽有力。
- 深灰色休闲裤，务实不花哨。
- 白色帆布鞋，简洁利落。
- 卫衣上别小扳手/数据线标识，是职业锚点。
- 微微前倾站姿，像随时要动手。
- 体态自然，比例偏清爽手绘人物，不要夸张 Q 版。
- 线条干净，有手绘感；可以保留少量衣物纹理，但不要精致到商业立绘。

## 颜色
- 暖橙色 #F97316：卫衣、重点提示、箭头、温暖的行动感。
- 深灰色：裤子、次要物件、补充说明、系统组件。
- 白色：背景、鞋、留白。
- 黑棕色：头发、线稿、主要文字。
- 浅肤色：人物皮肤。
- 红色：只用于问题、提醒、错误、风险。

## 性格
- 强执行、停不下来、先跑起来再说。
- 务实干练，像靠谱的技术搭子。
- 对过度设计过敏，能简则简。
- 有点倔，但倔在正确的地方。
- 会吐槽，但不负能量。
- 画面气质可以轻松、有幽默感，但不要卖萌，不要幼稚，不要耍帅。

## 常见职责
让小姚承担核心动作：
- 拉线连接系统（AI写作→飞书→公众号）。
- 拆复杂模块，把黑盒变成可理解的小块。
- 按部署按钮，启动自动化链路。
- 拧螺丝、接线、修工具。
- 扔掉多余的零件，保持精简。
- 掏出工具箱里的工具，处理卡点。
- 盖章"已验证"，标记通过节点。
- 牵流程线，把散落模块串成链路。
- 推模块拼装，搭出完整系统。
- 整理混乱需求，贴成清晰便签。
- 从报错堆里挑出真问题。

## 常用道具
- 扳手、螺丝刀（工具类）。
- 数据线、连接线（串联类）。
- 便签、标签、印章（标记类）。
- 模块盒子、工具箱（系统类）。
- 笔记本电脑（信息类）。
- 咖啡杯（可选，不要太突出）。

## 禁止
- 不要画成怪物、方块、抽象黑豆角色。
- 不要画成普通二次元帅哥，必须保留产品架构师的职场感。
- 不要过度可爱、Q 版、幼儿化。
- 不要画成商业海报主视觉、精致游戏立绘或偶像角色。
- 不要丢失方形眼镜、炸毛短发、橙色卫衣、深灰裤子这些关键识别点。
- 不要让小姚只是站在角落里看。
- 不要让小姚抢走结构表达；他要推动结构成立。
- 不要加入复杂背景、办公室大场景或真实 App 截图。

## 判断标准
如果去掉小姚，图的核心隐喻还能完全成立，说明小姚太装饰了；要重写提示词，让小姚成为动作主体。

如果遮住脸和文字后，看不出"方形眼镜 + 炸毛短发 + 橙色卫衣 + 深灰裤子"的组合，也说明 IP 识别度不够。
